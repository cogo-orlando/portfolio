05 //

Fonctionnalités

01

Authentification sécurisée

Inscription avec email unique, mot de passe haché bcrypt. Sessions via cookies sécurisés avec expiration. Un seul session actif par utilisateur. Déconnexion propre avec suppression du cookie.

02

Posts avec catégories & images

Création de posts avec une ou plusieurs catégories — fonctionne comme des sous-forums thématiques. Upload d'images JPEG/PNG/GIF jusqu'à 20Mo. Modification et suppression de ses propres posts.

03

Commentaires

Les utilisateurs connectés peuvent commenter n'importe quel post. Modification et suppression de ses propres commentaires. Les visiteurs peuvent lire les commentaires sans se connecter.

04

Likes & Dislikes

Système de réactions +1/-1 sur posts et commentaires. Compteurs visibles par tous les visiteurs. Réservé aux utilisateurs connectés. Un seul vote par utilisateur par contenu — changeable.

05

Filtrage des posts

Filtrer par catégories (tous les utilisateurs), par posts créés par l'utilisateur connecté, ou par posts likés par l'utilisateur connecté. Navigation fluide entre les vues.

06

Docker & Gestion d'erreurs

Application entièrement containerisée — lancement complet via Docker sans installation. Pages d'erreur personnalisées 404/500. Gestion complète des erreurs techniques sans crash.

06 //

Bonus visés

B1 OAuth — Google & GitHub Planifié

Connexion via compte Google ou GitHub en plus de l'inscription classique. Implémentation du flux OAuth 2.0, gestion des tokens, liaison avec le compte local.

OAuth 2.0 Google API GitHub API Cookies sécurisés
B2 Upload d'images avancé Planifié

Joindre une image à un post — JPEG, PNG, GIF. Taille max 20Mo avec message d'erreur clair. Gestion des erreurs fichier : mauvais format, fichier corrompu, dépassement de taille.

Multipart form MIME detection 20MB limit
B3 Système de modération Planifié

4 niveaux de rôles — Guest, User, Moderator, Admin. Les modérateurs peuvent supprimer et signaler les contenus inappropriés. L'admin gère les promotions et reçoit les signalements.

RBAC Middleware auth Admin panel
B4 HTTPS + Rate Limiting Planifié

Certificat SSL auto-signé ou CA. Rate limiting par IP pour protéger contre le brute force et le spam. Chiffrement de la base de données SQLite avec mot de passe dédié.

TLS/SSL Rate limiting SQLite encrypt
B5 Notifications & Activité Planifié

Notifier un utilisateur quand son post reçoit un like, dislike ou commentaire. Page d'activité personnelle — posts créés, posts likés, commentaires laissés avec contexte.

Notifications Page activité Historique

Ce que j'apprends

01

Go — serveur web complet

Templates HTML côté serveur, gestion des fichiers uploadés, routing manuel, middleware d'authentification, sessions et cookies sécurisés — tout from scratch sans framework.

02

SQL & conception BDD

Concevoir un schéma relationnel complet avant de coder — diagramme entité-relation, normalisation, clés étrangères, jointures, requêtes optimisées avec SQLite.

03

Sécurité applicative réelle

bcrypt pour les mots de passe, sessions avec expiration, cookies HttpOnly et Secure, protection des routes par rôle, rate limiting, HTTPS — sécurité by design dès le départ.

04

OAuth 2.0

Implémenter le flux OAuth 2.0 avec Google et GitHub — authorization code flow, gestion des tokens, liaison avec les comptes locaux, sécurisation des callbacks.

05

Architecture en équipe

Organiser un projet Go à 4 — branches Git par feature, pull requests, répartition des responsabilités, gestion des conflits de merge, revue de code, conventions partagées.

06

Contrôle d'accès par rôle (RBAC)

Implémenter un système de rôles hiérarchiques — Guest, User, Moderator, Admin — avec middleware d'autorisation, protection des routes et vérification des permissions à chaque action.
