Ce que j'ai appris

01

Architecture serveur Go

Construire un serveur HTTP from scratch sans framework m'a forcé à comprendre vraiment comment fonctionne le web — routing, middleware chain, template cache, graceful shutdown. Pas de magie, chaque ligne a une raison d'être.

02

Conteneurisation Docker

Le multi-stage build m'a appris que la taille d'une image n'est pas une fatalité — passer de 350MB à 60MB en séparant la compilation du runtime. Et qu'un process ne devrait jamais tourner en root, même en local.

03

Sécurité web concrète

La théorie c'est bien, la pratique c'est mieux. Implémenter un honeypot, du rate limiting, un panic recovery et des headers de sécurité m'a montré que la sécurité c'est une couche par couche — pas un interrupteur on/off.

04

Cloudflare & reverse proxy

Comprendre ce que fait vraiment Cloudflare — WAF, DDoS, SSL termination, cache — m'a donné une vision claire de ce qui se passe entre un navigateur et un serveur. L'infrastructure n'est pas une boîte noire.

05

CI/CD & tests automatiques

Écrire 47 tests unitaires et les faire tourner automatiquement à chaque push avec GitHub Actions m'a convaincu que les tests ne sont pas une contrainte — c'est ce qui te permet de modifier du code sans avoir peur de tout casser.

06

Logs & observabilité

slog JSON structuré, request ID, métriques /health — j'ai compris que sans logs exploitables, un serveur en production c'est une boîte noire. Savoir ce qui se passe en temps réel change complètement la façon de déboguer.
