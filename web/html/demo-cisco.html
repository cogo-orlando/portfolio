Réseau
Multi-Sites

Architecture réseau d'entreprise répartie sur 4 sites internationaux — conçue et configurée intégralement sous Cisco Packet Tracer. Routage statique, VLANs, DHCP sur Switch L3, redondance WiFi, DNS et serveur Web.

03 //

Architecture technique

Routage statique

Routes statiques configurées sur chaque routeur de site vers Router0 (hub central). Interconnexion WAN via sous-réseaux 10.0.0.x dédiés entre chaque site et le routeur central.

VLANs & Segmentation

10 VLANs répartis sur les 4 sites (VLAN10 à VLAN100). Chaque VLAN correspond à un segment réseau isolé avec son propre adressage IP. Routage inter-VLAN assuré par les switchs L3 (3650-24PS).

DHCP sur Switch L3

Le service DHCP est configuré directement sur les switchs Layer 3 (3650-24PS) de chaque site. Chaque VLAN dispose de son propre pool DHCP, distribuant automatiquement les adresses IP aux hôtes.

DNS & Serveur Web

Serveur DNS (192.168.100.10) et serveur Web (192.168.100.20) hébergés sur le site de Stockholm. Accessibles depuis tous les sites via le routage inter-sites configuré.

Redondance WiFi

Chaque point d'accès WiFi est doublé par un AP redondant sur tous les sites. En cas de défaillance d'un AP, la connectivité sans fil est maintenue via l'AP de secours sur le même VLAN.

Interconnexion WAN

Architecture hub-and-spoke avec Router0 comme nœud central. 4 liaisons WAN dédiées : 10.0.0.1↔2 (Strasbourg), 10.0.0.5↔6 (Bruxelles), 10.0.0.9↔10 (Casablanca), 10.0.0.13↔14 (Stockholm).
