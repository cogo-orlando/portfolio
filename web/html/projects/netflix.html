Fonctionnalités

01

Catalogue TMDB complet

Tendances du moment, films populaires, mieux notés, par genre. Film aléatoire en hero à chaque visite. Données réelles en temps réel depuis l'API TMDB — millions de films disponibles.

02

Authentification & Sessions

Inscription email/pseudo/mot de passe, connexion, déconnexion. Sessions gérées côté serveur Express avec middleware de protection des routes. Persistance dans users.json.

03

Détail film complet

Synopsis, casting complet avec photos, note TMDB, genres, durée, date de sortie. Films similaires recommandés. Interface immersive avec fond dynamique basé sur le poster du film.

04

Recherche fluide

Barre de recherche sur la page films et dans les favoris. Résultats dynamiques via l'API TMDB sans rechargement de page. Routing côté client géré manuellement via l'API History.

05

Système de favoris

Ajout et suppression de films depuis n'importe quelle page. Stockage par utilisateur dans favorites.json côté serveur. Synchronisé en temps réel avec le compte connecté.

06

Lecteur bande-annonce

Player vidéo custom construit sur l'élément HTML5 natif — contrôles natifs désactivés, remplacés par une interface JavaScript : barre de progression, play/pause, volume, timer, plein écran.

Choix techniques

Routing client manuel History API

Le routing côté client est géré manuellement via l'API History du navigateur — sans React, Vue ou autre framework. Navigation SPA complète sans rechargement de page.

Architecture découplée logic/routes/security

La logique métier (logic/) est séparée des routes Express (routes/) et du middleware d'authentification (security/). Chaque couche a une responsabilité unique et claire.

Stockage JSON Sans base de données

Les données utilisateurs et favoris sont stockées dans des fichiers JSON lus et écrits via un service dédié (services/storage.js). Choix délibéré pour simplifier le setup sans BDD.

Proxy TMDB côté serveur Clé API sécurisée

La clé API TMDB est conservée dans .env côté serveur et jamais exposée au navigateur. Le frontend communique exclusivement avec l'API Express, qui joue le rôle de proxy vers TMDB.

Player vidéo custom HTML5 natif

Le lecteur de bandes-annonces est construit sur l'élément video HTML5 — contrôles natifs désactivés, interface entièrement personnalisée en JavaScript : progression, volume, timer, plein écran.

Déploiement Render Live

Application déployée sur Render avec variables d'environnement sécurisées. Accessible à netflixlight.onrender.com — premier projet JavaScript solo déployé en production.

Ce que j'ai appris

01

Intégration d'API REST tierce

Consommer l'API TMDB depuis un proxy Express — fetch, gestion des réponses JSON, pagination, gestion des erreurs réseau et des cas limites. Clé API sécurisée côté serveur uniquement.

02

Architecture SPA sans framework

Construire une Single Page Application complète en JavaScript vanilla — routing avec History API, gestion de l'état global, mise à jour dynamique du DOM sans rechargement de page.

03

Backend Express.js structuré

Organiser un backend Node.js en couches séparées — routes, logique métier, middleware, services. Authentification par sessions, protection des routes, proxy vers API externe.

04

Player vidéo custom HTML5

Construire un lecteur vidéo complet from scratch — désactiver les contrôles natifs, implémenter la progression, play/pause, volume, timer et plein écran en JavaScript pur.
