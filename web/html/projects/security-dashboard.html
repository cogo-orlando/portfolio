Contexte

Mon portfolio est déployé en production sur orlandocogo.com — il reçoit du trafic réel, des bots, des scans automatiques et des tentatives d'accès à des routes sensibles. J'avais besoin d'un outil pour superviser ce qui se passe réellement sur mon infrastructure.

Le Security Dashboard est né de ce besoin concret — ce n'est pas un projet fictif, c'est un outil que j'utilise vraiment pour surveiller mon portfolio en temps réel.

"Je supervise et sécurise ma propre infrastructure — chaque ligne de code a une utilité réelle."

Fonctionnalités

01

Vue d'ensemble

KPI temps réel — total événements, requêtes 24h, honeypots déclenchés, IPs blacklistées. Graphique trafic par heure, top IPs et top paths avec barres de progression. Tendances ↑↓ comparées aux 24h précédentes.

02

Événements

Liste des 100 dernières requêtes avec filtres par type (honeypot, erreur, rate limit, brute force) et recherche en temps réel par IP ou path. Export CSV des 7 derniers jours.

03

Blacklist

IPs automatiquement blacklistées 24h après avoir déclenché un honeypot ou tenté une attaque brute force. Date de création et d'expiration affichées.

04

Trafic

Flux des 20 dernières requêtes en temps réel — IP, méthode HTTP, path, status code, type d'événement. Auto-refresh toutes les 15 secondes.

05

User Agents

Répartition des navigateurs et bots — Chrome, Firefox, Bot, curl, Python, Go HTTP. Cards avec barres de progression et graphique donut interactif canvas.

06

Erreurs groupées

Top 20 erreurs 4xx/5xx groupées par path et status code. Barres visuelles proportionnelles et indicateur de niveau (Critique / Attention).

07

Métriques système

Données live depuis le runtime Go — goroutines actives, mémoire allouée, cycles GC, uptime, version Go, OS, architecture. Tableau infrastructure complet.

08

Security Score

Score de sécurité 90/100 basé sur 10 critères — HTTPS, headers, rate limiting, honeypot, bcrypt, cookies, WAF, Docker hardening, DB sécurisée, alertes.

Ce que j'ai appris

01

PostgreSQL en production

Concevoir un schéma DB, activer RLS, créer des utilisateurs avec droits limités, forcer SSL — pas juste "connecter une DB" mais la sécuriser correctement pour la production.

02

Sécurité applicative concrète

Implémenter bcrypt, sessions sécurisées, rate limiting, headers HTTP — et corriger les vrais problèmes détectés par gosec. La sécurité c'est couche par couche, pas un paramètre.

03

Architecture multi-services

Faire communiquer deux applications Go distinctes via une base de données partagée, avec des utilisateurs et des droits différents selon le service — principe du moindre privilège en pratique.

04

Tests unitaires Go

Écrire 28 tests pour auth et middleware — tester les sessions expirées, le rate limiting, les cookies, les panics. Les tests ne sont pas une contrainte, c'est ce qui permet de modifier sans casser.

05

Charts canvas from scratch

Construire un graphique en barres et un donut chart sans librairie externe — canvas API, DPR, gradients, légendes. Comprendre ce que font vraiment Chart.js et D3 sous le capot.

06

Monitoring d'infrastructure réelle

Superviser un système en production — pas des données simulées mais de vrais événements, de vraies IPs, de vrais bots. Comprendre ce que signifie "observabilité" quand c'est ton propre serveur.
